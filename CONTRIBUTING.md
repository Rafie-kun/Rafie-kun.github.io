# Contributing

Thanks for helping improve this project.

## How to fork and run locally

1. Fork the repository.
2. Clone your fork if you want to work locally.
3. Open `index.html` or `editor.html` directly in a browser.
4. No server is required, and no setup commands are needed.

## Code style

- Use vanilla JavaScript only
- Do not introduce frameworks
- Keep the Minecraft pixel theme intact
- Preserve the existing fonts, colors, borders, shadows, and spacing style
- Keep changes simple and readable

## How to open issues

If you find a bug or have an idea, open an issue and include:

- What happened
- What you expected
- Steps to reproduce
- Browser and operating system

## How to submit pull requests

1. Open an issue first if the change is significant.
2. Create a branch in your fork.
3. Make your changes.
4. Test in the browser.
5. Submit a PR with a clear description of what changed.

## Design reminder

Any new feature should preserve the pixel art aesthetic, including the `Press Start 2P` and `VT323` font pairing, the grass/dirt/stone palette, and the blocky panel style.

## Testing reminder

Please test on Chrome, Firefox, and mobile before submitting.
